# 2026-05-18

> Project: FieldQuanta/predict_molecule · Tags: #paper-align #infra-optim #roadmap

- 中期 eval 跟 paper force MAE 仍有 gap, 仔细 re-read paper 角落发现 80-epoch run 用了 Gaussian RBF distance bias, 之前我们忽略了
- 启用 `use_rbf_bias=true` (TransIP 字面实现) 试图复现
  - rank 22 fp32 + `create_graph=True` **OOM at 78.97 / 80 GiB** (max_atoms=2000)
  - 根因: 8 个 attention layer + LDE 各自独立 `GaussianRBF`, 每层重算 `[B,M,M,64]` fp32 张量, autograd 二阶图翻倍 → 9 个大张量同时驻留
- 实现 `use_shared_rbf_bias=true` 优化路径
  - 相同拓扑 (per-layer `Linear(64, H, bias=False)`, **全表达力保留**, 等价于 paper 写的 single-layer RBF)
  - RBF expand 在 backbone forward 做一次, LDE + 8 attention layer 共享同一张展开结果
  - 实测单卡 GPU 显存 M=2000 fp32 + create_graph: **50.36 GB → 13.15 GB** (~4× saving)
- 严重怀疑 paper 实现真实性
  - paper 自称 80-ep run 是 single-layer RBF + fp32 + autograd forces, 按字面拓扑应该跟我们一样 OOM
  - 他们没 OOM, 可能因为: (a) 内部就是 shared 实现 (论文没写) / (b) fp32 + autograd 不是全程 / (c) 实际 max_atoms 远小于 2000 / (d) 用了更大显存的卡
  - 论文细节缺失, 一个关键优化没写, 复现成本高
- 用 shared-RBF 重启 strict-alignment 训练, 跑满 80 epoch 看最终能不能对齐
