# 2026-06-01

> Project: FieldQuanta/predict_molecule · Tags: #eval-result #decision #infra

- 消融结果出来了, 提升都很小 (最好的 sandwich 也就 +0.008)。 想清楚根因: 我们靠"把晶体转各种角度喂"(数据增广) 教模型旋转对称, 但这有天花板 —— 模型为了对转动不敏感, 干脆把方向信息全扔了, 退化成只看距离这种标量。 所以加方向类特征基本没用, 这是范式本身的上限, 小修小补突破不了
- 要突破得换思路, 不靠增广、改成先把晶体"摆正"再喂。 今天实现两条:
  - polar frame: 用晶胞自己算一个标准朝向, 把整个晶体旋到这个朝向再进模型, 模型就不必再学对称、能放心用方向。 数学上精确旋转不变, smoke 验证到机器精度 (转任意角度能量分毫不差)
  - frac-RoPE: 借大语言模型的位置编码思路, 让注意力直接读出原子间真实相对位置, 且不扭曲几何
- 实现 polar frame 时顺手揪出一个潜伏 bug: stress (晶格受力) 之前算法不对、不满足旋转对称, 修了。 会让 stress 数值变 (跟修复前不能直接比), 但能量和原子受力不受影响
- 两条都做成开关默认关、CPU smoke 通过, 写好消融脚本: base / polar / rope / 两者合体 四格对比, 卡一释放就跑, 主看能不能破 +0.008 这个天花板
- 想透一串晶格"规范"的边界问题: 模拟里晶格啥时候会动 (变胞 MD 和结构弛豫时被应力推, 打榜那步就是), 打榜弛豫怎么保证不出岔 (标准流程本就不重新约化, 我们不主动插即可), 极端固固相变/融化怎么办。 都记进 ROADMAP
- 查了文献确认方向对: 类似思路有人做过 (FAENet 给晶体做 frame, Crystalformer 做倒易空间注意力, 我们 base 的 TransIP 自己就证明了非增广比增广好)。 我们几个细节 (用晶胞极分解定朝向、RoPE 形式、basis 增广替约化) 这轮检索没撞到现成的
- 四格消融 frac-RoPE 那格先出了中间结果, 力预测大幅超 base: 跑到 ep21 force cosine 0.3214, base 同期 0.2972, 高 0.024, 是之前最好的 sandwich (+0.008) 三倍, 头一个真破天花板的。 polar 那格只小幅超, 合体还在跑。 这反过来印证瓶颈不在"对称怎么处理"而在"几何特征喂得够不够强": 把相对位置乘进注意力权重 (RoPE 这种乘性) 赢得多, 单纯加个方向标量 (polar 那些弱通道) 赢得少。 所以下一步就去做更强的乘性耦合
- 再往前一步, 现在整个网络已经做成完全旋转不变了。 立项时的设计是"非等变 + 数据增广", 靠转各种角度喂硬教对称; 现在把模型里仅剩的两个跟朝向有关的输入 (原子的绝对坐标、晶胞的原始矢量) 都拿掉, 换成转晶体也不变的量 (分数坐标、原子间距离、晶格形状描述子、frac-RoPE), 剩下全程不带朝向, 模型天生旋转不变、连增广都不用了, smoke 验能量精确不变。 等于从增广范式整个挪到"天生不变"这一侧 (跟姊妹项目 crystalite 同阵营, 但我们的几何特征不扭曲、它的分数坐标扭曲)。 挂了纯不变 baseline + 不变带 RoPE 两格, 看原来那套朝向机器到底是帮忙还是帮倒忙
- 实现第三条路线, 强耦合那条。 之前只把原子间距离喂给模型, 现在给每个原子算一个局部角度指纹 (周围邻居张成什么形状, 直线还是弯还是四面体), 再拿这指纹逐层去乘性调制模型内部特征。 加性喂 (把特征拼上去) 跟乘性调 (当旋钮拧每个通道) 差一个量级, 是扩散模型 DiT 的看家做法借过来。 默认关, smoke 验旋转不变到机器精度
- 实现时撞到一个坑顺手定了取舍。 这角度指纹吃方向, 晶体是周期的, 原子过半个晶胞时最近的那个周期镜像会突然换一个, 方向翻 180 度, 指纹值硬跳一下力就出尖峰。 严格修法是把所有周期镜像求和平均, 但那样显存爆 (125 份中间量全挂在求导图里, 大结构直接 OOM)。 想清楚打榜只跑结构弛豫不跑长程动力学, 这跳点落在一个几乎踩不到的几何面上、对最终能量没影响, 所以默认用便宜版 (直接复用已经算好的最近距离, 显存跟 base 一样), 严格版做成开关留着。 挂了消融跟 base 比, 先看这角度指纹到底有没有用
- 跑满 30 epoch 的几格消融结果出来了, 把"摆正/不变"这条路坐实了: 纯不变版跟 base 基本打平 (说明原来那套增广 + 绝对坐标根本没买到东西、白背), 而纯不变 + frac-RoPE 是目前全场最佳, 力预测一致性比 base 高 0.06, 是之前小修小补天花板 (0.008) 的七倍, 四项指标全赢。 angle 指纹那格 (geom_adaln) 还在跑, 但同期看力已经大涨, 能量暂时落后、等跑完才能定。 一句话主线结论: 瓶颈在"几何喂得够不够强", 不在"对称怎么处理"
- 顺这个结论今天把"强耦合"往深挖, 多做了两种把几何乘进内容的办法, 都默认关、smoke 过、挂了消融:
  - 低秩双线性: adaLN 的升级版。 adaLN 只能拿几何逐通道放大缩小每个原子的特征, 双线性能让几何在不同通道之间重新搭配内容。 数学上 adaLN 就是它的退化特例。 这是不上等变网络那套重机器、去逼近它"把几何和内容乘在一起"那个数据高效的根
  - 几何门控 FFN: 借大模型 SwiGLU 的门控思路, 在 FFN 内部拿几何当阀门调隐层
- 想清一件容易混的事: 这几种耦合"正不正交"看的是注入位置、不是形式。 RoPE 改注意力权重、adaLN 和双线性改输入内容、门控 FFN 改 FFN 隐层 —— 三个位置不同所以正交、可以叠; 但 adaLN 和双线性是同一个位置 (双线性已含 adaLN), 别当两个模块堆。 三个位置叠满, 大致就逼近等变网络那种耦合强度了
- 把这三条强耦合 (adaLN / 双线性 / 门控 FFN) 各自单独 + 配 RoPE, 合成一个七格统一消融 chain, 同一个 job 串跑保证完全同条件, 主看哪个注入位置最值、双线性比对角 adaLN 强多少
