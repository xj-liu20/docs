# 2026-06-09

> Project: FieldQuanta/predict_molecule · Tags: #eval-result #decision

- OAM 精调多训 2 个 epoch (2→4), 打榜 F1 (材料稳定性) 从 **0.587** 跳到 **0.761** (初步, 60% 测试集弛豫完, 最终 ±0.01)。 印证了昨天的判断: 昨天那版 2ep 是欠训, 留了个统一的能量偏置 (把所有东西往"稳定"判), 要靠事后给预测平移一个常数才看出排序其实学对了 (校准能涨 **0.085**); 今天 4ep 这个偏置训没了 (校准只剩 **0.005**), 原始就 0.761。 不是模型不行, 是没训够。 能量误差也从 0.068 降到 **0.040**, 砍一半; R2 从 0.638 升到 0.776。

- 这个数的份量: 同一套评测下 eSEN-3.2M 是 0.568, 我们纯 MPtrj 最好 (29M) 0.643, 现在 **17M 的 OAM 一个 4ep 冲到 0.761**, 逼近 eSEN-30M (人家 30M, 0.819)。 正好验证"MPtrj 上加参数没用、OMat24 大数据上加训练才兑现"那条 —— 纯加参数 F1 不动, 这里加 epoch 直接 +0.17。 非等变 + 大数据替代等变这条路, 第一次拿到强证据。 更大的 M (37M) 还在弛豫, 预期更高。

- 实现了两个长程相互作用模块 (默认关), 治之前发现的卤化物 (离子晶体) 被过度判稳的毛病 —— 那类缺的是长程静电 (马德隆能)。 之前试过一版长程直接爆了 (训练发散), 这次搞清楚了根因: 老版本把一个"全局平均"塞进每个原子, 既非局域 (一个怪原子污染全体能量) 又带除法奇点。 新两版都是数学上不会爆的形式 —— 一个 (EFA) 带距离衰减的核读进特征, 一个 (LES) 把电荷做成单独的静电能量项 (两两求和天生广延)。 冒烟全过, 默认关不动现有训练。

- 初步测了长程有没有用 (训练集内, 快): LES 把卤化物的误差富集从 **1.30 降到 1.15** (方向对、微弱), EFA 没帮上。 但这个探针偏短程、量错了地方 (长程修的是能量和分布外, 不是训练集内的力), 真正裁决等分布外榜单 F1, 已挂上跑。 一个有意思的小结论: 受约束的物理形式 (LES) 比更灵活的 EFA 反而有信号 —— 灵活但跟现有全局注意力冗余、又没指向正确物理, 就学不到东西, 跟"等变 (约束) 打败灵活"是同一个道理。

- 更大的 M (37M) 也弛豫完了: F1 **0.778** > S 的 17M 0.761, 能量误差 0.041 也更低。 这正是要的对照 —— MPtrj 上加参数 F1 不动甚至降, 换到 OMat24 大数据上加参数就涨了 (+0.017)。 = "参数和数据要一起涨" (Chinchilla): 小数据堆参数撞地板, 大数据上加参数才兑现。 MAE 已逼到 0.04, 离 eSEN-30M 的 0.032 不远。

- 修了个反常的 OOM: 大模型 (L/XL, 63M/97M) 的 OAM 微调显存爆, 但同尺寸的预训练不爆。 根因是多数据集微调挂了两个能量头 (MPtrj 一个、sAlex 一个), 每个都单独算一次力的二阶导、占双倍显存, 大模型就翻过上限。 改成一个头同时算两个数据集的能量、只算一次二阶导, 显存砍半。 顺带修了个潜在 bug: 之前两个头互相覆盖力的结果, MPtrj 结构拿到的是 sAlex 头算的力 (能量没事、F1 没受大影响, 但力是错的)。 验证全过 (路由后的力跟参考逐位相等、端到端微调+推理都通)。 试过更省事的 grad checkpointing 但实测没用 (二阶导那条路它省不掉显存)。

- 长程那两个模块的分布外榜单 F1 出来了, 基本白做: base17 **0.638**, EFA **0.641** (噪声), LES **0.640**。 误判稳定的比例三个一模一样 (9.7%)。 昨天在训练集内看 LES 把卤化物力误差从 1.30 降到 1.15 以为有戏, 结果那个探针量错地方了 —— 短程的力不等于分布外的稳定性排序。

- 跑 worst case 才看清, 挺有意思: LES 不但没治好卤化物, 反而把卤化物的假阳性富集从 **1.34 搞到 1.51** (更糟)。 因为 LES 的电荷是在 MPtrj 上学出来的、不是固定的物理静电修正, 分布内拿它多拟合点力, 但学到的电荷到分布外卤化物上不管用, 等于过拟合。 跟 EFA 没用一个道理: 加的是灵活但没正确物理约束的自由度, 拟合训练集、伤分布外。

- 但同一个 worst case 给了个更强的正结果: 卤化物这老毛病 **OAM 数据直接治好了** (富集 base17 1.34, OAM-M 降到 **0.97**, 不再过表达), LES 想做没做到的数据免费做到还更彻底, 整体过度判稳的偏置也砍半。 worst case 迁移到 chalcogenide (S/Se/Te) 和 f 区 (镧系锕系) 了, 前者像表征问题、后者是 DFT 标签本身难。 结论: 长程这条收了, 到 0.9 还是靠数据加拉长 OMat24 训练。

- 修了 XL finetune 的 OOM: XL 预训练在 8000 原子能跑 (薄余量), 但二阶段多数据集 finetune 还是爆。 单 grad 那个修复是生效的, 是 finetune 多挂的几个小头加中间量把薄余量顶过 80G 了。 给脚本加了个旋钮单独压 finetune 阶段的 batch (XL 用 4000, S/M 不传就还是 8000、零影响), 重启拿现成的预训练 ckpt 跳过预训练就行。 不过 M 已经 0.778 了, XL 同数据预期也就再 +0.01 上下, 跑完当 scaling 端点看, 不指望它冲 0.9。

- 揪出一个挺隐蔽的回归, 顺带更正昨天那条"OOM 根治"。 排查 XL 力曲线为啥平的时候发现: 之前为修 L/XL 显存改的那个多数据集合并头, 按数据集名路由能量, 但它比的是 "mptrj" 这个裸名, 而运行时实际的名字带后缀是 "mptrj.train"/"mptrj.val" —— 永远对不上, 等于路由出来的能量恒为 0, 结果力和应力从训练第一步起就没拿到梯度、压根没在学 (能量没事, 它在路由之前算)。 XL 的力 loss 一直卡在 3.5 (= 预测全零时目标力的幅值), 旧双头的 S 是 1.3 往 0.97 降。 改一行 (路由比名字时去掉后缀) 同时治好训练和评测显示。 只有用这个合并头的 XL 中招, 旧双头的 S/M/L 力都正常、F1 真实。 XL 这版要重训。 定位是顺着"评测只显示 mptrj、能量正常但力是平的"这个线索摸到的。
