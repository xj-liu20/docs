# 2026-06-18

> Project: FieldQuanta/predict_molecule · Tags: #result #experiment

- 6ep 重地基的完整 F1 出来了,unique 0.8224 (full 0.8071),比 4ep 的 0.8151 涨 +0.0073,已经超过 eSEN-30M-MPtrj 的 0.819。重地基这条阶梯 2→4ep 是 +0.0148,4→6ep +0.0073,大致每 2ep 减半,递减但还没停,按这趋势 8ep 估再 +0.003 左右。加 epoch 还能榨一点,但性价比在降,大杠杆得转到架构和放大了。

- 踩了个坑得记一下:relax pool 没跑完就拿前期快照算 F1,被带偏了。partial 采样一路 0.8211(52%)→ 0.8181(81%)→ 0.8224(完整),摆动 ±0.004。原因是 stride 分片下没跑完的样本成分是偏的,慢的那俩 shard 覆盖的成分偏有利,跑完才回正。中间我一度根据 81% 的 0.818 说"6ep 摸顶只 +0.003",是错的。结论:partial-pool 前期数不可信,只能当极粗方向,要数就等完整 8-shard。

- 边通道 reallocation(把径向容量从 attention bias 挪到边通道,srbf 64→8 配 edge 8→32/64)梯度爆炸,edge 降到 32 还炸。诊断清楚了:边通道的 edge state 是跨层 recurrent 更新的(e=e+update,过 8 层),attention bias 是算一次 broadcast 不迭代,所以同参数下边通道梯度增长快得多;加边径向就是加宽这个 recurrent 路径,把本来就在临界点的 lr=0.01 推过去(baseline edge=8 本来就偶发回退,贴着边)。srbf=8 是我误抓的 confound,边宽才是因。修法两条在权衡:降 lr 到 0.003 加 base 同 lr 重跑(不改码但要配着拉 step 否则欠训、且跟 0.8151 不可比),或给 edge state 加个 norm 保住 lr=0.01(改码但根上治、省 step、保可比)。

- 顺手把当前 nmlpsh 架构各 size 的真实参数量实测了一遍,之前用的是 6/12 边通道之前的旧数(那是父类计数,小了 1.4×)。当前 S 20.8M / M 44.6M / L 77.5M / XL 119.3M,170M 对应 d=1152,不是之前以为的 d=1320。还实测了 170M 的显存天花板:pretrain max_atoms 8000 OOM,6000 能跑但贴边(78GB),安全档 5000,finetune 更重还没测。结论 170M 能跑,不用动架构,只要 max_atoms 设 5000,代价是 throughput 降三成、步数变多,不丢结构。
