# 2026-06-27

> Project: FieldQuanta/predict_molecule · Tags: #experiment #result #decision

- 把四个 recipe 的下游全测齐了,拼出一张 κ-F1 权衡表:baseline(PS)κ1.54/F1 0.822、noPS@0.00025 κ1.31/F1 0.813、lean κ1.38/F1 0.811、freeze-8ep(正常 lr 训透)κ1.41/F1 0.836。规律很干净:训得越透,F1 和上游力/能量越好(freeze 的 F1 0.836 连 baseline 都超了),但 κ 越差。算 CPS,欠训的 noPS@0.00025 还是最优(0.545,比 baseline +0.041),freeze 第二(0.536,F1 的涨几乎补平了 κ 的亏)。所以训练量本身就是 κ 和 F1 的权衡轴。
- 这个权衡的机制是"力-曲率过拟合",不是经典那种。upstream val 的力/能量是随训练变好的(所以不是 train/val 过拟合),但没人监督的曲率(κ 是能量二三阶导,我们从不直接训 Hessian)在退化——猛优化力目标会伤声子。这正是 eSEN 说的 test error 不是物性的可靠代理,我们用欠训/中/透训三档实证了一遍。欠训其实是"意外把曲率正则住了"(脆弱巧合,不是真解);PS 当年就是个显式的曲率正则,删了才暴露这个过拟合。
- 由此定下一步:既然 0.00025-欠训(κ好F1差)和 8ep-透训(F1好κ差)是同一条权衡曲线的两端,就用 2ep + freeze + 激进 lr 去中间扫甜点——比 0.00025 训多让 F1 涨、比 8ep 训少让 κ 保住,两边都占一点,CPS 有机会破 0.545。挂了 lean 和 noPS 两个 2ep run(freeze 0.2ep 给 head 预热、warmup 0.3ep 必须大于 freeze 才有 backbone 平滑接入)。真正解耦 κ-F1 还得靠曲率正则,2ep 只是先把 operating point 摸对。
- infra:给 eval 的 pool(kappa/relax)加了 GPU 列表支持,设 GPUS=0,1,3,4,5,6 就能跳过被别的 job 占满 100% util 的 GPU2(default off 不影响别的)。freeze 的 F1 因为有个 shard 卡在 GPU2 上慢 3×,手动拆成 3 路重跑到空闲卡才收尾。
- 2ep 实验补充:激进 lr 0.004 连 freeze 都压不住,step 600 就崩,回退到 0.001 才训完。说明曲率悬崖把 finetune lr 上限钉在 ~0.001,freeze 只削掉新 head 冲击那一截、削不掉悬崖本身——又一次指向真解是曲率正则不是 lr/freeze 这种 trick(到 0.001 就到顶)。两个 2ep run(lean/noPS,都 0.001+freeze)训完了,κ/F1 还在测(watcher 之前盯错了崩掉的 0.004 那俩,已切到 0.001)。
- 另开了条架构线:noPS + 把边通道从 64 加宽到 128(我们 node:edge 是 1:6,PET/ORB/eSEN 都 1:1,边太窄),完整重训 pretrain 6ep + finetune 2ep。edge128 显存大,max_atoms 从 8000 降到 5000 + 开 split_linear 才塞下 DDP(8000 会 OOM)。顺手把 freeze 改成只作用 finetune——之前全链下那个全局 env 会连 pretrain 从头训也一起冻,不对。这条测加宽边帮不帮 κ,跟曲率正则那条并行推,大概 1 天出。
- 2ep 实验出了第一个结果,是个清晰的负结果。lean_2ep(bias16+noPS,2ep):κ 1.2875 最佳、101/103 有效声子(虚频几乎清零),但 F1 崩到 ~0.70(partial 65% 实算,不是估的)→ CPS ~0.494,比 baseline 还低、目前最差。根因是 2 个 epoch 能量头根本没收敛(上游 E/atom 0.042 比 8ep 的 0.022 差一倍),而 F1=稳定性=形成能,跟能量精度走、不是力——我先按力外推估 0.80,实算 0.70,差了 0.1,上游外推不靠谱。所以 κ-F1 权衡是硬的、2ep 冲过头到了 κ极好/F1极差那端,往训少调 F1 掉得比 κ 涨得快,operating-point 这个旋钮扫到头也破不了 0.545(当前最优还是 noPS@0.00025)。
- 一个重要的概念修正:之前说"准曲率必须直接监督 FC2/FC3"是错的,ORB 反例打回——ORB κ0.21 没加任何声子数据、纯能量/力/应力就训出准曲率。所以差距不在监督,在容量:ORB 边≈节点 512-1024、15 层、局部 MP 为主,我们边 64(1:6)、8 层、全局 attention 为主。κ-F1 权衡其实是我们容量不足的症状、不是物理定律(ORB 容量够就没这权衡,训透同时拿 κ 和 F1)。所以破 CPS 的唯一路是把容量提上去(加宽边 + MP 为主),不是接着调训练量,也不是去喂声子数据。edge128 那两条就是判决实验。
