# 2026-06-27

> Project: FieldQuanta/predict_molecule · Tags: #experiment #result #decision

- 把四个 recipe 的下游全测齐了,拼出一张 κ-F1 权衡表:baseline(PS)κ1.54/F1 0.822、noPS@0.00025 κ1.31/F1 0.813、lean κ1.38/F1 0.811、freeze-8ep(正常 lr 训透)κ1.41/F1 0.836。规律很干净:训得越透,F1 和上游力/能量越好(freeze 的 F1 0.836 连 baseline 都超了),但 κ 越差。算 CPS,欠训的 noPS@0.00025 还是最优(0.545,比 baseline +0.041),freeze 第二(0.536,F1 的涨几乎补平了 κ 的亏)。所以训练量本身就是 κ 和 F1 的权衡轴。
- 这个权衡的机制是"力-曲率过拟合",不是经典那种。upstream val 的力/能量是随训练变好的(所以不是 train/val 过拟合),但没人监督的曲率(κ 是能量二三阶导,我们从不直接训 Hessian)在退化——猛优化力目标会伤声子。这正是 eSEN 说的 test error 不是物性的可靠代理,我们用欠训/中/透训三档实证了一遍。欠训其实是"意外把曲率正则住了"(脆弱巧合,不是真解);PS 当年就是个显式的曲率正则,删了才暴露这个过拟合。
- 由此定下一步:既然 0.00025-欠训(κ好F1差)和 8ep-透训(F1好κ差)是同一条权衡曲线的两端,就用 2ep + freeze + 激进 lr 去中间扫甜点——比 0.00025 训多让 F1 涨、比 8ep 训少让 κ 保住,两边都占一点,CPS 有机会破 0.545。挂了 lean 和 noPS 两个 2ep run(freeze 0.2ep 给 head 预热、warmup 0.3ep 必须大于 freeze 才有 backbone 平滑接入)。真正解耦 κ-F1 还得靠曲率正则,2ep 只是先把 operating point 摸对。
- infra:给 eval 的 pool(kappa/relax)加了 GPU 列表支持,设 GPUS=0,1,3,4,5,6 就能跳过被别的 job 占满 100% util 的 GPU2(default off 不影响别的)。freeze 的 F1 因为有个 shard 卡在 GPU2 上慢 3×,手动拆成 3 路重跑到空闲卡才收尾。
- 2ep 实验补充:激进 lr 0.004 连 freeze 都压不住,step 600 就崩,回退到 0.001 才训完。说明曲率悬崖把 finetune lr 上限钉在 ~0.001,freeze 只削掉新 head 冲击那一截、削不掉悬崖本身——又一次指向真解是曲率正则不是 lr/freeze 这种 trick(到 0.001 就到顶)。两个 2ep run(lean/noPS,都 0.001+freeze)训完了,κ/F1 还在测(watcher 之前盯错了崩掉的 0.004 那俩,已切到 0.001)。
- 另开了条架构线:noPS + 把边通道从 64 加宽到 128(我们 node:edge 是 1:6,PET/ORB/eSEN 都 1:1,边太窄),完整重训 pretrain 6ep + finetune 2ep。edge128 显存大,max_atoms 从 8000 降到 5000 + 开 split_linear 才塞下 DDP(8000 会 OOM)。顺手把 freeze 改成只作用 finetune——之前全链下那个全局 env 会连 pretrain 从头训也一起冻,不对。这条测加宽边帮不帮 κ,跟曲率正则那条并行推,大概 1 天出。
- 2ep 实验出了第一个结果,是个清晰的负结果。lean_2ep(bias16+noPS,2ep):κ 1.2875 最佳、101/103 有效声子(虚频几乎清零),但 F1 崩到 ~0.70(partial 65% 实算,不是估的)→ CPS ~0.494,比 baseline 还低、目前最差。根因是 2 个 epoch 能量头根本没收敛(上游 E/atom 0.042 比 8ep 的 0.022 差一倍),而 F1=稳定性=形成能,跟能量精度走、不是力——我先按力外推估 0.80,实算 0.70,差了 0.1,上游外推不靠谱。所以 κ-F1 权衡是硬的、2ep 冲过头到了 κ极好/F1极差那端,往训少调 F1 掉得比 κ 涨得快,operating-point 这个旋钮扫到头也破不了 0.545(当前最优还是 noPS@0.00025)。
- 一个重要的概念修正:之前说"准曲率必须直接监督 FC2/FC3"是错的,ORB 反例打回——ORB κ0.21 没加任何声子数据、纯能量/力/应力就训出准曲率。所以差距不在监督,在容量:ORB 边≈节点 512-1024、15 层、局部 MP 为主,我们边 64(1:6)、8 层、全局 attention 为主。κ-F1 权衡其实是我们容量不足的症状、不是物理定律(ORB 容量够就没这权衡,训透同时拿 κ 和 F1)。所以破 CPS 的唯一路是把容量提上去(加宽边 + MP 为主),不是接着调训练量,也不是去喂声子数据。edge128 那两条就是判决实验。
- 测了 pretrain(只 OMat24 预训练、没做 MPtrj+sAlex finetune)的 κ,1.054,是所有 recipe 里最好的。把 finetune 量排开 κ 完全单调:pretrain 1.054 < noPS_2ep 1.134 < lean_2ep 1.288 < noPS@0.00025 1.31 < lean-8ep 1.38 < freeze-8ep 1.41 < baseline 1.54,finetune 越多 κ 越差。
- 一度怀疑这是不是 finetune 数据或单位 bug,查了一遍排除掉。MPtrj aselmdb 的 stress 实测 0.0002-0.003 eV/Å³(不是 GPa,GPa 会大 160 倍)、force 0-0.4 eV/Å、E/atom -2~-8,全在物理范围,1.42M 条没坏(文档里那个 lmdb_evpa3 现在没了,换成别人 5/27 建的 aselmdb,单位已验对)。更硬的判据是 finetune 把有监督的力和能量都训得更好(上游 E/atom 0.042→0.022,力也降),只有没监督的曲率在退化——数据坏的话有监督指标也会烂,但它们在变好。所以不是 bug,是力-曲率过拟合加分布收窄:OMat24 离平衡、力多样最能约束曲率,finetune 收到 MPtrj+sAlex 这个更窄的分布上,优化器把力压得更准曲率就飘软了,跟文献近平衡软化和 PEFT 防遗忘是一回事。
- 所以 pretrain 到 finetune 这个策略本身没错,榜上模型都这么干,问题是我们容量不够、被逼在 κ 和 F1 之间二选一,ORB/eSEN 同样 OAM 数据容量够就两个都拿、没有这个权衡。finetune 不是坏了,是对 κ 次优。
- debias 试了,用 eval_ours_adapted 重拟合 per-element 形成能 refs(89 参数)。2ep 的 raw F1 0.70 debias 后到 0.80,涨 0.09,所以 2ep 的 F1 崩盘大半是可校正的逐元素能量偏置。但意外的是所有模型 debias 后都收敛到 0.79-0.80,训透的反而掉(baseline 0.822→0.791、freeze 0.836→0.792),欠训的涨。含义很硬,raw F1 从 0.70 到 0.836 这一段差异几乎全是 per-element convention 偏置、不是真稳定性排序能力,我们架构内禀的 F1 天花板就是 0.80。之前一直追的那点 F1 差大半是假象。
- 两个连起来,debias 拉平 F1 后 κ 成了 CPS 唯一区分项(κ 是 reference-independent、debias 碰不到)。adapted 口径下 operating-point 最优从 8ep 翻到短训,noPS_2ep 的 CPS_adapt 0.573 断层第一(raw 才 0.531),pretrain+debias 若 F1 真到 0.80 则 0.589。caveat 是 adapted 拟合在 WBM 测试集上、等于泄漏、是上界,leaderboard 用 raw 那边 noPS@0.00025 0.544 还是最优;合规版要在 held-out 拟合 per-element 校正再套 WBM,判决测试就是 WBM 切半、A 半拟合 B 半算,B 仍 0.80 就能上榜。pretrain 的 F1 现在 every20 全量在测,它能不能 debias 到 0.80 决定 pretrain+debias 这条路成不成。
- pretrain F1 出来了,raw F1_uniq 0.540 惨(R2 -0.25,能量 convention 离 MP2020 极远),adapted 到 0.784。差一点没到 0.80,但 κ 优势够大,adapted-CPS 0.581 仍是第一。WBM 弛豫比我估的快太多,12849 结构 16 并行 269 struct/min、48 分钟就跑完了,不是我先前说的 6-7 小时,估错了一个量级。
- 然后把 debias 合不合法这个命门测了,这是整条路的判决。写了个 5-fold 交叉验证脚本 eval_ours_debias_cv.py,每折用其余折拟合 per-element refs、预测留出的那折,每个点都拿到校正器没见过它时的 held-out 校正,跑 pretrain / noPS_2ep / noPS@0.00025 三个。结果很干净:pretrain raw 0.540、held-out CV 0.7795、fit-on-test 0.788,held-out 几乎追平 fit-on-test、回收 97%;noPS_2ep 0.715→0.8011(fit-on-test 0.802,回收 99%)。held-out 约等于 fit-on-test 就说明这个 per-element 偏置是真系统性、跨化学空间泛化,不是 fit-on-test 过拟合,debias 上榜合法。
- noPS@0.00025 这种训透的反过来,raw 0.8128 比 CV 0.8075 还高、debias 反而微伤,因为它能量 convention 本就对齐。坐实了 debias 只救 convention 没对齐的欠训模型这个判断。
- 所以合规 CPS 用 held-out F1 算:pretrain+debias 0.5·0.7795+0.4·0.473=0.579 新最优,noPS_2ep+debias 0.574,noPS@0.00025 不 debias 0.544。两条新路都比之前最优 0.545 高 0.03 上下,operating-point 最优真的翻案了,零或极少 finetune 加 debias 比适量 finetune 不 debias 强。
- 但 pretrain 0.579 是靠 0.245 的巨额 debias 撑起来的、极度依赖合法性,noPS_2ep 0.574 只靠 0.085、raw 本就 0.715、真 skill 也更高,更稳,所以务实首选是 noPS_2ep+debias、pretrain 当激进上限。还有两个没量的风险要补:一是 RMSD(CPS 占 10%、我们从没测过),pretrain 没在近平衡训过、弛豫到平衡的几何可能差、会拖 CPS,可能让 2ep 反超;二是这个 CV 是在 WBM 内切折,最干净的上榜版应该在非 WBM 的 OAM val 上拟合 per-element 校正再套 WBM,CV 已经证了机制、非 WBM 拟合几乎必然也成,但要做实得再跑一次。
