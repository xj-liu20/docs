> Project: FieldQuanta/predict_molecule · Tags: #experiment #result #decision

- lean_edge256 那个 κ 0.638 重跑确认了,第二次跑出来还是 0.6376、99/103,完全复现,不是 pipeline 抖动。加上之前分布验证(系统变硬贴 DFT、非退化),这个突破实锤,CPS ~0.68 站得住,是目前项目最大的结果。
- 顺着"窄距离 bias"这条轴又试了 bias8 @ edge256,结果 overshoot 了。edge256 上把 attention 距离 bias 的 RBF 通道扫一遍是个 U 形:bias64 是 1.244、bias16 是 0.638、bias8 回到 0.7348。窄化帮到 16 就到头,再窄到 8 太粗、attention 分不出距离了,κ 反而退回去。所以 bias16 就是 edge256 的 κ 甜点,不用再窄(bias4 没意义)。F1 还在测,但 κ 已经定案 bias8 不如 bias16。
- 由此定下一步方向:窄-bias 这条轴探明了(16 最优),剩下的杠杆是把边通道加宽往 1:1 走(edge384)。真正想要的是 edge384 配甜点 bias16。
- 一个命名要记一下,免得后面搞混:我挂的 lean_edge384 这个 run 名字骗人,它真实 config 是 edge_dim 384 加 shared_rbf_num_channels 8,也就是 edge384 + bias8,不是 edge384 + bias16。edge:node 到了 384:384 也就是 1:1,正好是 ORB 那个目标比例,但 bias 配的是次优的 8。所以它出来若比 bias8_edge256 好,说明 edge384 那步在帮;若要这条线的最优还得补一个 edge384 + bias16。
