# 2026-07-04

> Project: FieldQuanta/predict_molecule · Tags: #experiment #result #decision

- edge384(实际是 edge384 + bias8,边:节点到了 1:1)测完了,κ_SRME 0.5770、held-out F1 0.8458、CPS 0.708,三项都是新最优,比之前最好的 lean_edge256(κ0.638/F1 0.822/CPS 0.683)全面高。这是继 lean_edge256 之后第二个里程碑。
- 最关键的是它把 κ-F1 权衡打破了。这是个干净的单变量对比:edge256+bias8 到 edge384+bias8,只把边加宽到 1:1,结果 κ 从 0.735 降到 0.577、held-out F1 从 0.823 涨到 0.846,两个一起变好。之前所有杠杆(训练量、bias 宽度、freeze)都是在权衡线上滑,一个好另一个就亏;加宽边到 1:1 是第一个让 κ 和 F1 同时涨的,正是我一直说的容量分配假说——边:节点到 1:1、容量分配对了,就像 ORB 那样没有权衡。这条"重分配到边、往 1:1"的架构线兑现了。
- 而且这还是用的次优 bias8(在 edge256 上 bias16 才是甜点)。正在训的 edge384+bias16 可能更高,不过 bias16×edge 那个交互说宽 edge 上 bias 甜点会往窄移,所以 bias8 在 edge384 上说不定已经接近最优了,等 edge384+bias16 出来才好判。
- lmax4 也 smoke 过了。edge_sh_lmax 从 3 加到 4(角度分辨率 SH 16→25 通道),跑了 test_edge_channel 验证,build、step0 恒等、双反向梯度有限加边通道参数拿到梯度、不卡零初始死锁,全 PASS,守恒力二阶导没岔子。所以可以在 edge384+bias16 上再叠这个角度轴。
- 大局:榜首 CPS 0.89,我们从最初的 0.585 一路到 0.708,缺口从 0.30 砍到 0.18,走了近一半,主要靠架构线(重分配到边到 1:1、窄距离 bias、debias 救 F1、少 finetune 保 κ)。下一步 edge384+bias16、+lmax4,以及可能的 edge512。
