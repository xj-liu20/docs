# 2026-07-08

> Project: FieldQuanta/predict_molecule · Tags: #experiment #result #decision

- M(edge576,全模型 54.9M)完整三项 CPS 出来了,约 0.786,比 S(edge384,24.5M)的 0.757 涨 0.029。三项:F1_uniq 0.8582、κ_SRME 0.4654(102/103 材料出有限 κ)、RMSD 0.0757。最大功劳是 κ:S 是 0.577、M 压到 0.4654,κ 占 CPS 40% 权重、一直是最弱项,这次降了一大截。容量轴从 S 放大到 M 是 κ 和 F1 一起涨、不是只涨一项,验证了 scaling 假说。离榜首还有距离:ORB-v3 0.888 / eSEN-30M-OAM 0.918,M 把 S 的缺口从 0.13 缩到约 0.10。
- L(dim768,97M)挂上了,max_atoms 按前一天实测定的 pretrain 2500 / finetune 2000。但 pretrain 中途 step 1500(epoch 0.07)就 OOM 了。原因是之前那个显存实测是用 sub 小数据跑的、峰值只到 55GB,全量 OMAT 有更大的晶胞结构,真数据一上来峰值冲过 80GB。所以 2500 对 L pretrain 在全量数据上还是不够,得再往下压 max_atoms 或者动架构。
- 顺带确认了个之前的疑点:之前给 attention 加的分块省显存开关(attn_chunk_size)在晶体这条线上基本没用。它砍的是注意力的 [B,头,M,M] 矩阵,但晶体是把很多小结构打包进一个 batch、M 等于 batch 里最大那个单结构(几百个原子),不是 max_atoms 那个总预算,所以注意力矩阵本来就小、砍它省不下什么。真正吃显存的是随总原子数涨、又挂在守恒力二阶图上的东西:边通道、PS 全镜像几何、以及 create_graph 的二阶导图,这几个那个开关都不碰。那个开关是给分子那条线(OMol25,一个大分子 max_atoms 就是序列长)设计的,晶体这边瓶颈不在注意力。
- 真实环境消融实测了 L 显存到底哪块最大:边通道占 90%(关掉从 50.7 掉到 4.8 GiB),而且它也是算力大头(M=64 注意力几乎为零)。普通 grad checkpoint 在保守力二阶导下确实没用,还写了个最小例子实测确认:二阶导下开 checkpoint 峰值 1.06→1.28 反而涨(重算的前向激活被二阶图钉住释放不掉)。真正的解是照 attention 那个 chunked SDPA 套路,给边通道写了分块双反向的自定义 autograd.Function(edge_chunk_size flag,默认关):把边核在 E 维度分块、两次 backward 都按块重算。验证前向+一阶梯度+二阶导 param grad 全部 ~1e-7 匹配;显存 max_atoms=2000 时 50.7→12.25 GiB(4×);L 在 max_atoms=5000 从 OOM 变成能跑(pretrain/finetune 32/43 GiB,真训练两 stage 都跑通)。提交了,默认关不影响现有。
- 但测了时间代价:边通道被重算,每步慢 2.7-2.9×,而且大 batch 抵消不了——测了 no-chunk 的 µs/原子从 max_atoms 800 到 2000 是持平的,说明模型已经算力饱和、没有小 batch 利用率低的空档给大 batch 填。对晶体这种小结构,max_atoms 只是打包预算、每 epoch 总算力不变,所以 no-chunk@max_atoms=2000 满速反而吞吐最优。分块这个能力是给单结构本身就大(大分子/超大晶胞,没有降 max_atoms 的退路)或想腾显存塞更大模型用的,先存档。
- 修了个 lr 默认坑:run_omat24 脚本 pretrain/finetune lr 默认还挂着旧的 0.01/0.004,但 S 和 M 实际都用 0.005/0.001。查出来之前挂的那个 L run 没传 override、吃了 stale 0.01,97M 从头训偏高(loss 乱跳 16-20 大概率跟这有关)。把默认改成 0.005/0.001(WARMUP_EP_FT 那个既有参数化也保留),重挂 L 自动就对了。L 建议 no-chunk、max_atoms=2000 重挂。
