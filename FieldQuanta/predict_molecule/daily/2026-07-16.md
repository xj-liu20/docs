# 2026-07-16

> Project: FieldQuanta/predict_molecule · Tags: #experiment #decision

- 昨天 S µP 三档 matbench 出来发现 CPS 都比无 µP 老基线低一点、差主要在 κ,当时留了个假说说是 embedding 移桶惹的。今天把 keller→moonshot→µP 的 CPS 下降翻实际 metrics.json 拆成三步定了案。keller κ 0.577 / F1_uniq 0.844 / CPS 0.708;换成 moonshot(还没上 µP)κ 0.568 / F1 0.831 / CPS 0.702;再加 µP 移桶 κ 冲到 0.61-0.66 / CPS 0.687-0.693。
- 结论是 κ 的大头亏损落在移桶那一步(moonshot→µP),不是 moonshot 本身。moonshot 单独看 κ 其实还略好一点(0.577→0.568)、F1 略差、CPS 只小降 0.006。之前口头记的"单纯 moonshot κ 降 F1 升"方向反了,实际是 κ 略好 F1 略差,不过 CPS 降这个方向是对的。
- 有个省事的发现:S 上移桶对照根本不用重跑。S 的 µP width-scale factor 是 1.0、跟非 µP 字节等价,所以已经跑完的 plain moonshot 和 µP 这俩 run 本来就只差移桶一件事,它俩就是现成的移桶 A/B。原本想挂的那个 mup_embed_adamw=false 的 S 对照跑出来会跟 plain moonshot 字节等价,纯重复,砍了。
- 既然坏的是移桶、好的是 width-scale(缩 hidden lr、放大时治 skip、还该帮 κ),那就拆开用:放大时保留 width-scale、只把移桶关掉。挂了个 M 的对照验这个(TAG=_noembedmove,mup_embed_adamw=false,其余全 µP 默认)。相对现有 µP-M(移桶开、κ 0.727 / F1_uniq 0.863)只差移桶,在 M 上干净隔离;同时对比无 µP 老 M 的上千次 skip,看 width-scale 保留能不能给稳定性。要是 κ 能回到 0.57 档且训练稳,就是鱼和熊掌都要,定成 L/XL 默认配方。也顺手修了原来 XL µP 默认把移桶开着的隐患。
